Глава 7.1. Промисы

При написании асинхронного кода в JavaScript, чтобы выполнить какое-то действие после завершения асинхронной операции, применяют функции обратного вызова или, проще говоря, колбэки. Идея функций обратного вызова предельно проста и удобна в использовании, но с колбэками легко случайно свернуть на шоссе «ад колбэков».

Что такое «ад колбэков»?

Код на JavaScript, в котором используется много-много вложенных функций обратного вызова. Так много, что в таком коде трудно интуитивно разобраться. Больше информации и примеры на callbackhell.ru

Бороться с «адом колбэков» можно несколькими способами. Самый банальный — реструктуризация кода. Отказаться от пары колбэков. Функции, что останутся, вынести и записать в переменные. И таким образом немного разгрузить основной код. Способ рабочий, но не сильно действенный, потому что провернуть такой трюк получается далеко не всегда.

Более элегантный способ заключается в применении Promise(с англ. «обещание». Чаще не переводится, поэтому далее «промис»). Поддержка промисов появилась в ECMAScript 2015. Промисы позволяют организовать асинхронный код по-другому: сделать его более плоским и последовательным, похожим на синхронный код.

Promise (промис)

Промис — это глобальный объект Promise. Он позволяет отследить выполнение асинхронной операции и сохранить её результат. Сами асинхронные операции выполняются как и прежде: не сразу, а когда-нибудь. Поэтому важно запомнить и уяснить, промисы — это не «магическая» замена асинхронных операций и не возможность получить результат такой операции сразу. Это просто объект, позволяющий более эффективно работать с асинхронным кодом.

Синтаксис

Чтобы воспользоваться промисами, необходимо создать экземпляр объекта Promise с помощью new:

const myPromise = new Promise((resolve, reject) => {});

Аргументом в new Promise() нужно передать функцию. Её принято называть «функция-исполнитель» (от англ. executor). У этой функции два параметра-колбэка:

  • resolve — колбэк, который нужно вызвать, если промис завершён успешно;
  • reject — колбэк, который нужно вызвать, если промис завершён с ошибкой.

Например:

const myPromise = new Promise((resolve, reject) => {
  if (2 > 1) {
    // Если всё верно, вызываем resolve, и тем самым объявляем, что myPromise завершён успешно
    resolve();
  } else {
    // Если в какой-то вселенной 2 не больше 1, вызываем reject, и тем самым объявляем, что myPromise завершён с ошибкой
    reject();
  }
});

Что значит «завершён», и о других состояниях промиса мы поговорим в конце главы.

Важно уточнить, что функцию-исполнителя браузер выполнит сразу. Это может быть неочевидно на первый взгляд. Для примера запустите в консоли код:

const myPromise = new Promise(() => {
  console.log('Привет! Я функция-исполнитель.');
});

После выполнения в консоль сразу же выведется 'Привет! Я функция-исполнитель.'

Использование

Возникает закономерный вопрос «Где же асинхронность?» Внутри функции-исполнителя! Поскольку она вызывается сразу, асинхронный код встанет в очередь на выполнение, и останется только дождаться, когда он завершится, чтобы вызвать resolve или reject. Причём колбэк resolve должен быть вызван в случае успешного выполнения асинхронной операции, а reject — если что-то пошло не так.

Представим наш пример про приготовление завтрака (а конкретно каши) из главы про асинхронность в виде кода на промисах. Каша — это промис porridge. Асинхронной мультиваркой у нас выступит переменная multicooker, которой для простоты примера можно добавить обработчики событий done (каша готова) и error (что-то пошло не так):

const porridge = new Promise((resolve, reject) => {
  multicooker.addEventListener('done', () => {
    resolve('Овсянка, сэр.');
  });
  multicooker.addEventListener('error', () => {
    reject('Каши не будет, электричество кончилось.');
  });
});

// Другой код программы...

Если нам нужно вернуть значение, его нужно передать аргументом resolve или reject.

Функцию-исполнитель браузер выполнит сразу, и мультиварке multicooker добавятся два обработчика событий. Затем выполнение перейдёт к другому коду программы. Колбэки же resolve или reject будут вызваны лишь тогда, когда у мультиварки наступят соответствующие события. Вот она, асинхронность.

Результат

Всё складно, кроме одного. Если выполнение после создания промиса переходит к другому коду программы, как нам получить результат, который будет передан, например, в resolve? Для этого у каждого созданного промиса есть метод then():

const porridge = new Promise((resolve, reject) => {
  multicooker.addEventListener('done', () => {
    resolve('Овсянка, сэр.');
  });
  multicooker.addEventListener('error', () => {
    reject('Каши не будет, электричество кончилось.');
  });
});

+// В then первым аргументом мы передаём функцию,
+// которая станет тем самым колбэком resolve
+porridge.then((message) => {
+  console.log(message); // 'Овсянка, сэр.'
+});

// Другой код программы...

Вторым аргументом можно передать функцию, которая станет колбэком reject.

Таким образом выстраивается цепочка:

  • браузер создаёт промис porridge и выполняет функцию-исполнитель;
  • функция-исполнитель добавляет обработчики мультиварке multicooker с вызовом колбэков промиса;
  • далее выполняется переход к then(), и с его помощью мы указываем браузеру, что нужно сделать, когда от мультиварки будет результат (выполнить переданную в then() функцию);
  • при этом, пока нет результата, разрешаем браузеру выполнять другой код программы.

Метод then() своего рода мостик к функции-исполнителю, которую мы определили при создании нашего промиса porridge.

Промисы в реальной жизни

Ненадолго вернёмся к теории и более скрупулёзно посмотрим на промисы. Дословно с английского слово «promise» переводится как «обещание». Перевод отчасти объясняет основную концепцию промисов. При выполнении асинхронной операции нет возможности получить результат сразу. Вместо этого мы получаем обещание, что задача будет выполнена. Возможно успешно, а может быть с ошибкой. Об этом разработчик узнает, когда обещание будет выполнено.

Проведём аналогию с реальной жизнью. Представьте, что вы собрались в кинотеатр и покупаете для этого билет. Обычно билет в кино приобретается заблаговременно в интернете. И в данном случае билет является обещанием, что вы сможете посмотреть кино, но это произойдёт не сразу после покупки. Сперва нужно добраться до кинотеатра. И когда вы будете сидеть в удобном кресле и наслаждаться просмотром, можно сказать, что был вызван колбэк resolve и обещание выполнено... промис завершён успешно.

Но! Всегда есть место форс-мажору, и в назначенный час в кинотеатре могут отключить электричество. Кино не покажут, но вам обязательно об этом сообщат. В этом случае можно сказать, что был вызван колбэк reject и промис также завершён, но с ошибкой.

Состояние промиса

Мы уже с вами затрагивали состояние завершённости промиса, давайте разберёмся во всех возможных состояниях промисов.

Как только мы объявляем промис, он попадает в состояние pending, что означает выполняется:

const myPromise = new Promise(() => {});

Затем, когда и если будут вызваны колбэки resolve или reject, промис перейдёт в состояние settled, что означает завершён:

const myPromise = new Promise((resolve, reject) => {
  if (2 > 1) {
    resolve();
  } else {
    reject();
  }
});

Обратите внимание, состояние меняется автоматически в зависимости от вызываемого колбэка. Никакой специальный код, который явно будет менять состояние промиса, писать не нужно.

В зависимости от того, будет вызван колбэк resolve или reject, состояние завершённости соответственно может быть fulfilled, что означает завершён успешно, или rejected, что означает завершён с ошибкой.

Итого получается 3 состояния (settled не считаем за отдельное):

  • pending — выполняется
  • settled — завершён:
    • fulfilled — завершён успешно
    • rejected — завершён с ошибкой

Почему важно помнить о состояниях промиса

Промисы не позволяют откатиться на предыдущее состояние. Такой возможности не существует. Например, если мы вызвали resolve, переведя тем самым промис в состояние fulfilled, то даже если сразу после выполнить reject, состояние промиса не изменится:

const myPromise = new Promise((resolve, reject) => {
  resolve(); // На этой строчке myPromise будет успешно завершён 
  reject(); // И вызов reject уже ни на что не повлияет
});

Справедливо и обратное: после выполнения reject вызов resolve не приведёт к изменению состояния промиса.

Кстати, выполнение сразу и resolve, и reject не приведёт к ошибке, просто тот колбэк, который будет идти вторым, уже не повлияет на состояние промиса.

Поэтому будьте внимательны, когда вызываете resolve или reject. Обратного пути не будет.

Резюме

Мы познакомились с промисами и узнали, как их применять на практике. Всё, что для этого требуется — создать экземпляр объекта Promise. При использовании промисов стоит запомнить несколько важных вещей. Функция-исполнитель выполняется сразу. Промис может находиться в трёх состояниях. Нет возможности откатиться к предыдущему состоянию.

Промисы сложны в понимании, но они сильно упрощают применение асинхронных операций. По сути мы продолжаем использовать колбэки, но промисы позволяют это сделать более элегантным образом. В результате асинхронный код визуально становится более понятным и очевидным.
